# AI-Game-Builder 使用文档

本文档按「最短启动路径 -> 常用操作 -> 自检排错」整理，适合日常开发与联调。

## 1. 最短启动路径

### 1.1 启动后端

```bash
python server.py
```

服务地址：`http://localhost:8000`

### 1.2 启动前端

```bash
cd frontend
npm run dev
```

页面地址：`http://localhost:1420`

### 1.3 一键点亮舞台

1. 打开前端「系统设置」
2. 配置 `Godot 引擎绝对路径 (.exe)`
3. 点击「导出预设自检」
4. 三项全部 OK 后点击「启动 run_stage.py（点亮舞台）」
5. 中间区切到「实时画面」

## 2. 系统设置说明

系统设置支持以下参数：
- `GODOT_PATH`：Godot 可执行文件路径
- `Web 端口`：默认 `8060`
- `导出预设`：默认 `Web`
- `跳过导出`：仅启动 `web_build/` 静态服务

### 2.1 导出预设自检

按钮：`导出预设自检`

结果项含义：
- `GODOT_PATH OK / Missing`：Godot 路径是否存在
- `Export Preset OK / Missing`：`godot_project/export_presets.cfg` 中是否存在指定预设名
- `<端口> Listening OK / Not Listening`：本机端口是否已被监听

建议：
- 若前两项通过、端口未监听：点击「启动 run_stage.py（点亮舞台）」
- 若预设缺失：先在 Godot 编辑器 `Project -> Export` 新建 HTML5 预设

## 3. 界面分区

- 左侧：AI 控制舱（模式切换、模型配置、实体工坊、对话）
- 中间：舞台挂载区（网格占位 / 实时画面 / 热区点击）
- 右侧：世界设置 + 资产与属性面板

## 4. 三种工作模式

### 4.1 创意助理（chat）
- 适合剧情、关卡、数值、玩法讨论

### 4.2 实体工坊（build）
- 适合实体生成、组件增量更新、直连装配

### 4.3 美术中心（art）
- 适合素材生成与入库

## 5. 推荐工作流（视觉化组装）

1. 右侧素材库点选目标图片
2. 左侧工坊自动回填实体名与装配预览
3. 勾选组件
4. 点击「一键视觉化组装」

系统会走 `direct_assembly=true`，直接下发装配动作并写入存档。

## 6. 中间舞台区

### 6.1 网格占位与实时画面
- 网格占位：编辑阶段使用
- 实时画面：iframe 加载 `http://localhost:8060`（或你设置的端口）

### 6.2 状态灯
- `Live Link Active`：联动可用
- `Live Link Offline`：联动未建立

### 6.3 热区选中
- 基于 `scene_state.entities[].x/y` 映射热区
- 点击热区后联动左侧实体选择与属性展示

## 7. 世界设置

可配置项：
- `background_image`
- `background_color`
- `physics_gravity`

点击「应用世界配置」后：
- 后端持久化到 `stage_save.json.global_config`
- 广播 `update_scene_config` 给 Godot
- 前端控制台会打印下发 JSON 指令

## 8. 同步机制（当前版本）

- `GET /api/scene_state`：前端轮询同步场景状态（当前间隔 3 秒）
- WebSocket 心跳：前端每 5 秒发送 `ping`，后端返回 `pong`
- 心跳超时自动重连，避免假在线

## 9. 常见问题与修复

### 9.1 点亮舞台失败
- 先点「导出预设自检」
- 若 `GODOT_PATH Missing`：重新配置 `.exe`
- 若 `Export Preset Missing`：在 Godot 创建 HTML5 导出预设（名称与设置中的“导出预设”一致）
- 若端口未监听：执行点亮按钮或手动运行 `python run_stage.py`

### 9.2 实时画面白屏
- 检查设置中的端口与 iframe 地址是否一致
- 检查是否被防火墙拦截端口
- 点击中间区“重试连接”

### 9.3 Live Link 离线
- 确认后端 `8000` 在线
- 确认 Godot LiveLink 连接地址为 `ws://127.0.0.1:8000/ws`

### 9.4 前端构建报错
- 优先使用 Node 20/22 LTS
- 用 `npx vite build` 做构建验证

## 10. 常用接口速查

- `GET /health`：服务健康状态
- `GET /api/scene_state`：拉取场景状态
- `POST /api/scene/config`：下发世界设置
- `POST /api/generate_entity`：统一生成入口
- `GET /api/stage/precheck`：舞台自检
- `POST /api/stage/activate`：启动 run_stage.py
- `GET /api/stage/status`：查询舞台服务状态
